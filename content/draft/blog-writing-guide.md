# 博客写作指南

## 写作流程

### 1. 确定主题

动笔前回答一个问题：**这篇文章帮读者解决什么问题？**

- 主题要具体 — "React 19 的 3 个破坏性变更" 优于 "前端开发心得"
- 好的选题来源：你踩过的坑、解决的问题、踩坑后的经验总结

### 2. 拟定标题

标题决定点击率，先写草稿，写完正文再精修。

常用公式：

| 类型   | 示例                               |
| ------ | ---------------------------------- |
| 数字型 | `5 个技巧让 Next.js 构建更快`      |
| 问题型 | `为什么你的 CSS 优先级总是冲突？`  |
| 对比型 | `Zustand vs Jotai：状态管理怎么选` |
| 实操型 | `用 better-auth 接入 GitHub 登录`  |

### 3. 列大纲 ⭐

**最关键的一步**，大纲有了，写作就是填空。

```
1. 开头 — 什么问题，为什么值得看（2-3 句）
2. 背景 — 读者需要的前置知识
3. 正文
   3.1 第一部分
   3.2 第二部分
   3.3 第三部分
4. 注意事项 / 踩坑记录
5. 总结 — 3-5 条要点
```

### 4. 写正文

- **开头 3 行定生死** — 让读者知道能得到什么
- 段落要短，一句一意
- 用"你"而不是"我们"，像对话
- 技术文章：**先给代码，再解释**，别反过来
- 不要边写边改，先写完再改

### 5. 补充素材

- 代码片段必须**可运行**，别截半截
- 关键概念配图（架构图、流程图），一张图顶 500 字
- 附上参考链接，方便读者深入

### 6. 修改润色

- 大声读一遍，拗口的地方就是需要改的
- 删掉不服务读者的内容
- 此时再精修标题，确保标题和内容匹配

## Front Matter 格式

本项目的文章使用 MDX 格式：

```mdx
---
title: '文章标题'
description: '一句话摘要，用于 SEO 和卡片预览'
date: 2026-05-07
published: true
---

正文从这里开始...
```

- `published: false` 可隐藏文章而不删除
- `date` 决定排序（最新在前）
- `description` 会出现在 meta 标签和文章卡片中

## 常见误区

| 误区           | 正确做法             |
| -------------- | -------------------- |
| 一上来就写正文 | 先列大纲             |
| 追求一遍写好   | 先写完，再改         |
| 一篇写所有东西 | 一篇文章解决一个问题 |
| 标题最后才想   | 先定方向，写完再精修 |
| 不读自己的文章 | 放半天，自己读一遍   |

## 发布前检查

- [ ] 标题具体且有吸引力
- [ ] 开头说明了问题和你能获得什么
- [ ] 结构清晰（标题层级、逻辑连贯）
- [ ] 代码示例已测试、可运行
- [ ] Front Matter 字段完整正确
- [ ] 间隔一段时间后通读一遍
